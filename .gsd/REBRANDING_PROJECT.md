# Project: Rebranding & Firebase Stabilization

## Vision
Transform the project's identity from "Halo Bus" to "Halo Bus" across all platforms (Web, Mobile, Documentation) and stabilize the Firebase configuration which currently reports an invalid API key. Ensure that all security policies (Firebase Rules) from the test environment (Bannu Bus) are active in production.

## Tech Stack
- **Frontend**: React / Next.js
- **Mobile**: Flutter / Dart
- **Backend**: Node.js
- **Firebase**: Firestore, Authentication

## Non-negotiable Constraints
- No occurrence of the name "Halo Bus" should remain in UI, titles, or code strings.
- Firebase connectivity must be restored for both Web and Mobile.
- Existing functionality must not be broken during the rebranding.
